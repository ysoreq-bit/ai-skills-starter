# RESEARCH-001 — Multi-Source Synthesis

- **Scope:** Research workflow
- **Status:** Candidate
- **Trigger:** The user asks for a comparison, recommendation, investigation, landscape review, or report based on multiple sources.
- **Negative trigger:** Do not invoke the full workflow for a stable fact answerable from one authoritative source.

## 1. Define the target

State the decision or question, audience, time horizon, constraints, and required depth. Choose the output type: quick brief, focused summary, comparison, or comprehensive report.

## 2. Build a source ledger

For each source record:

- title and author or publisher;
- date and freshness;
- source type and authority;
- exact claim supported;
- limitations or conflicts;
- link, identifier, or repository path.

Prefer primary and authoritative sources. Use independent sources to test important claims and identify disagreement.

## 3. Search in focused passes

Use separate targeted queries for distinct questions. Do not rely on one broad query or search-result snippets when the source can be opened directly.

Stop expanding when additional sources no longer change the conclusion, reveal meaningful disagreement, or fill an important gap.

## 4. Synthesize

1. Group findings by the user's questions rather than by source.
2. Separate evidence, interpretation, inference, and recommendation.
3. Surface contradictions, missing data, version differences, and uncertainty.
4. Distinguish current information from stable historical background.
5. Keep recommendations tied to the user's real constraints.

## 5. Report

Include a concise conclusion, evidence-backed findings, trade-offs, risks, open questions, and actionable next steps when appropriate. Place citations next to the claims they support.

## 6. Capture

When research discovers a reusable source, tool, or method, send it through `META-001 — External Method and Tool Intake` before adding it to the active library.
